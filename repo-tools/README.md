# Package catalog tooling

Deploy config and scripts for the declarative JSON catalog and static site in
[`../repo/`](../repo/).

Static assets (no `/api` prefix):

- `../repo/repo.json` — package listing → `https://repo.pmil.me/repo.json`
- `../repo/package/<name>.json` — per-package install metadata (snake_case)
- `../repo/package.schema.json` — package-entry JSON Schema
- `../repo/repo.schema.json` — listing JSON Schema
- `../repo/index.html` — homepage (generated)
- `../repo/package/<name>.html` — package page → `/package/<name>` (generated)
- `../repo/site.css` / `../repo/site.js` — shared styles and copy-button script

Published as Cloudflare Workers static assets (see `wrangler.json`, assets
directory `../repo`) at `https://repo.pmil.me` via a Custom Domain route
(`routes` + `custom_domain: true`). `html_handling` is `drop-trailing-slash` so
`/package/caddy` serves `package/caddy.html`. The `pmil.me` zone must already be
on Cloudflare; deploy creates/updates the DNS record and certificate.

## Generate site

After adding or removing `../repo/package/<name>.json`:

```sh
deno task gen-site          # from repo root or repo-tools
```

This rewrites `repo.json` `packages` from the JSON files and regenerates HTML
(files marked `<!-- generated by repo-tools/scripts/gen-site.ts; do not edit -->`).

## Deploy (Deno only)

Follows the [Deno Wrangler tutorial](https://docs.deno.com/examples/cloudflare_workers_wrangler_tutorial/)
pattern (`deno add npm:wrangler`, no `package.json` / `node_modules`):

```sh
cd repo-tools
deno task deploy:dry   # gen-site + validate without uploading
deno task deploy       # gen-site + deploy to Cloudflare
deno task dev          # local preview
```

`../repo/` must stay static assets only — never put executable code or deploy
config there. Keep catalog edits under `../repo/package/` and `../repo/repo.json`.
